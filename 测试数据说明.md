## 🔧 数据管理

### 查看数据

**通过 API**:
```bash
# 查看 Banner
curl http://localhost:9000/api/banners

# 查看所有用户（需要管理员权限）
# 通过 phpMyAdmin 查看
```

**通过 phpMyAdmin**:
- 访问: http://localhost:9080
- 服务器: `mysql`
- 用户名: `root`
- 密码: `course123456`

### 添加更多数据

运行添加数据脚本：
```bash
docker-compose exec api node scripts/add-more-data.js
```

### 重置数据

如果需要清空并重新初始化数据：

```bash
# 停止服务并删除数据
docker-compose down -v

# 重新启动
docker-compose up -d

# 初始化数据
docker-compose exec api node scripts/init-data.js

# 添加更多数据
docker-compose exec api node scripts/add-more-data.js
```

## 📱 测试建议

### 测试不同用户

由于每个用户只能看到自己的数据，建议：

1. **测试用户隔离**:
   - 使用不同的 openid 登录
   - 查看各自的课程和记录

2. **测试数据丰富度**:
   - 不同用户有不同数量的课程
   - 不同课程有不同数量的记录
   - 记录时间分布在不同的日期

3. **测试 Banner**:
   - 首页应该显示 8 条 Banner
   - Banner 按 sort 字段排序
